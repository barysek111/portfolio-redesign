## Anti-Patterns Verdict

Not AI-generated. Distinctive choices: 3D blob hero, SVG principles map, photo slideshow. Two cautions: service card grid (three identical icon+heading+body cards, saved by distinctive icons/inverted palette), and locked-in chapter formula across all case studies. CLI detector unavailable.

## Overall Impression

A technically strong portfolio with a clear aesthetic POV. The single biggest problem is that the contact form is a silent dead end. The second biggest: "How I Work" appears before the actual work on the homepage.

## What's Working

1. Hero is genuinely distinctive — 3D blob + typographic intro is not the default UX portfolio move.
2. "How I Work" SVG principles map is a purpose-built component, not a template.
3. Visual consistency — token system holds throughout; one component vocabulary.

## Priority Issues

**[P0] Contact form sends nothing** — handleSubmit calls setSent(true) and resets. No fetch, no email service, no form action. Every submission silently discarded. Fix: wire Formspree or Cloudflare Worker.

**[P1] Nav has no active page indicator** — Renders identically on every route. No aria-current, no active class. Fix: read current route with useLocation() and apply active state to matching nav item.

**[P1] Case studies have no chapter navigation** — 6-7 chapters per case study, no anchor links or sticky progress. Fix: horizontal strip of chapter pills (scroll-to anchors) at top of content.

**[P2] Homepage content order buries the portfolio** — Flow: hero → How I Work → Project showcase. Fix: swap to hero → Project showcase → How I Work.

**[P2] Project showcase hover model on touch/mobile** — CSS :has(.hover-zone:hover) reveals content; touch devices don't emit hover. Fix: verify mobile state; ensure info is always visible on touch.
